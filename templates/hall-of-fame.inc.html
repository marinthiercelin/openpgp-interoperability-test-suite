Hall of Fame

The OpenPGP interoperability test suite found the following bugs:

Sequoia

  • sq: Impossible to generate key with multiple userids
  • attempt to subtract with overflow if signature is larger than expected
  • Blowfish encryption is broken
  • Marker packet not ignored in certificate parser
  • Expiration of primary key binding signatures is not honored
  • Sequoia does not honor certificate expiration on primary userid when iterating over subkeys

DKGPG

  • Lowercase hex digits are not recognized as such
  • Keyrings must be armored
  • dkg-encrypt adds spurious newline
  • Decryption fails if the size of the session key is not 256 bits
  • Generating a key with multiple userids silently ignores all but first userid
  • dkg-encrypt prefers OCB over the recipients preference for EAX
  • dkg-sign adds spurious newline to armored output
  • dkg-verify fails to verify signatures with leading zeros
  • Verification fails with "key was not intended for signing" though it is
  • dkgpg excludes trailing whitespace from hashing when verifying text signatures
  • Unknown notations with the "critical" bit set SHOULD invalidate the signature
  • dkgpg fails to enforce packet boundaries
  • dkgpg fails to decrypt message with unknown packet versions
  • dkgpg fails to verify detached signatures with unknown packet versions
  • dkgpg should ignore marker packets when verifying detached signatures
  • dkgpg throws std::bad_alloc on duplicated subkeys

GopenPGP

  • Expiration of primary key binding signatures is not honored
  • GopenPGP fails to decrypt message with unknown packet versions
  • GopenPGP fails to verify detached signatures with unknown packet versions
  • GopenPGP should ignore unbound or unknown certificate components
  • GopenPGP does not honor certificate expiration
  • GopenPGP successfully verifies signatures using SHA-1

OpenPGP.js

  • Unknown notations with the "critical" bit set SHOULD invalidate the signature
  • Signatures without signature creation time subpackets are invalid
  • Expiration of primary key binding signatures is not honored
  • OpenPGPjs should ignore marker packets when verifying detached signatures
  • OpenPGPjs does not honor certificate expiration on direct key signatures

RNP

  • Generating a key with multiple userids silently ignores all but last userid
  • Unbounded recursion when decrypting messages leading to unbounded allocation
  • Unknown notations with the "critical" bit set SHOULD invalidate the signature
  • Expiration of primary key binding signatures is not honored
  • RNP fails to decrypt message with unknown packet versions
  • RNP fails to verify detached signatures with unknown packet versions
  • rnp --encrypt ignores --output
  • RNP does not honor certificate expiration on direct key signatures
  • RNP takes certificate expiration information from non-primary userid bindings
  • RNP successfully verifies signatures using SHA-1

PGPy

  • Primary key binding signature is not checked
  • Critical, unknown subpackets and notations SHOULD invalidate signatures
  • PGPy seems to ignore key flags
  • PGPy fails to decrypt message with unknown packet versions
  • PGPy fails to verify detached signatures with unknown packet versions
  • PGPy should ignore marker packets when verifying detached signatures
  • PGPy should ignore marker packets when parsing certificates
  • PGPy should ignore unbound certificate components
  • PGPy does not honor certificate expiration

GPGME

  • Dubious filename in literal data packet when encrypting with GPGME/GnuPGv1.4.23
  • GPGME gets FDs confused in high concurrency settings, deadlocks on select
  • GPGME reports an incorrect engine version if the path is not absolute
  • GPGME uses --with-keygrip unconditionally, breaking GnuPG 1.4

GnuPG

  • GnuPG creates keys that it cannot use, likely related to AEAD
  • GnuPG 1.4.23 ignores key flags
  • All versions of GnuPG fail to enforce packet boundaries
  • Expiration of primary key binding signatures is not honored
  • All versions of GnuPG fail to decrypt message with unknown packet versions
  • All versions of GnuPG fail to verify detached signatures with unknown packet versions
  • All versions of GnuPG fail to ignore signatures and keys with unknown packet versions in certificates
  • All versions of GnuPG successfully verify signatures using SHA-1
